Required text Input

Example

Developer notes

This is a standard text input field. This version is required for form submission, losing focus of the form field checks if the field has been completed, and reveals it if it hasn't.

To turn validation on, the js-validation wrapper needs the validate function ran on it (shown in JavaScript code panel). This is only needed once per form.

You can change the error message on by changing the data attribute on the field.

Usage notes

At the beginning of the form, it is necessary to clearly state that some form fields are required, and inform the user you are using an asterisk to mark these.

<div class="form__item form__item--string">
  <label class="form__label" for="ff-text-validate-1">Required field
    <span class="form__label__asterix">*</span>
  </label>
  <input class="form__field form__field--string" type="text" name="" placeholder="" id="ff-text-validate-1" data-error-message="Required field" required />
</div>

JavaScript

$('.js-validation').validate({
   'keyUpValidation': true
 });

Backend error

Example

Developer notes

Backend errors are shown by adding a class of "form__item--backend-error" to the appropriate "form__item". After interacting with an element this class is removed and the front end takes over.

<div class="form__item form__item--string form__item--backend-error">
  <label class="form__label" for="ff-text-backend-error">Backend error</label>
  <input class="form__field form__field--string" type="text" name="" placeholder="E.g. example"
  id="ff-text-backend-error" data-error-message="Required field"
  required />
</div>
